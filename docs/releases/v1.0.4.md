## 更新说明

### 连接和回传稳定性

- 修复 Windows 版在部分环境下缺少 `telnetlib` 导致无法连接的问题。
- 后端改为随项目携带 CPython 3.12 的 `telnetlib.py` 兼容层，不再依赖运行环境自带的 Python 标准库 `telnetlib`。
- 修复通过 Telnet/base64 回传配置结果时，因中途短暂停顿被误判为超时的问题；现在会明确等待结束标记后再完成回传。
- 增加 Telnet 连接重试，降低设备短时无响应或网络瞬断导致的失败率。

### 日志和隐私

- 增加安全的每步耗时日志，界面会显示连接、检查配置、解析、接收保存、清理等步骤耗时，以及接收数据大小。
- 优化 debug log：回传失败时不再写入原始 base64 配置片段，只记录字节数、标记状态等脱敏信息。

### macOS 和工程整理

- macOS 版改用标准 `Info.plist` 字段管理版本信息：`CFBundleShortVersionString` 和 `CFBundleVersion`。
- 移除 macOS 生成源码版本文件机制，避免构建后污染 Git 工作区。
- 重构 App 图标资产目录，macOS / Windows 构建均直接引用已提交的平台图标资产。
- 新增本地 release 归档到 NAS 的流程；GitHub Actions 明确跳过该本地归档步骤。

### 已验证

- Windows x64：build30 经过真实设备测试通过。
- macOS arm64：本地构建、签名校验和 DMG 校验通过。
- GitHub Actions 多平台构建通过。
