## 更新说明

这是 v1.0.4 之后围绕真实设备回传稳定性、日志/缓存路径和错误提示体验整理的一版。

### 后端流程和回传稳定性

- 步骤 3 解密完成后，前端会立即显示已解析出的超级管理员账号和密码，不再等完整 XML 保存流程结束后才展示。
- 步骤 4 优先只回传包含超级密码和设备信息的必要 `Dir` 块，减少 Telnet/base64 回传数据量，降低大文件传输导致的卡顿或断连概率。
- 过滤结果为空或设备侧 `awk` 兼容性异常时，会自动回退完整配置文件回传，避免因为过滤失败直接中断。
- 步骤 4 回传失败时支持 Telnet 重连重试 3 次，并使用 5s/10s/15s 退避。
- 回传重试时会重新执行复制和 base64 读取流程，避免断连后继续使用无效连接。

### 临时文件清理

- 失败路径也会尝试清理本次设备侧临时文件。
- 清理步骤断连时支持重连重试，重连成功后继续清理。
- 日志会显示具体要清理的临时文件名；重连失败会展示具体原因，三轮失败后才提示需要手动登录设备确认。

### macOS 路径整理

- macOS 缓存和日志迁移到标准 Application Support 目录：
  `~/Library/Application Support/com.jiulian.superpassword-tool/`
- Bundle ID 改为 `com.jiulian.superpassword-tool`，日志路径跟随 Bundle ID 动态生成。
- 后端 debug log 由前端显式传入 `Logs` 目录，不再散放在用户 Home 目录。

### 错误提示和 UI

- 所有失败信息统一使用错误级别，GUI 中以红色显示。
- 去掉部分冗余的“请重试”类兜底提示，优先保留具体步骤和具体错误原因。
- 调整过 macOS 结果框和日志框布局；最终保留稳定布局，同时保留标准日志路径等修复。

### 已验证

- 已通过本地 Python 语法检查。
- 已通过 Telnet mock server 验证登录、命令执行、大段 base64 回传和账号密码解析流程。
