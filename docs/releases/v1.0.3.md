## 更新说明

### 兼容性修复

- 修复 Windows 版在部分环境下提示缺少 `telnetlib`、导致无法连接光猫的问题。
- 后端改为随项目携带 CPython 3.12 的 `telnetlib.py` 兼容层，不再依赖运行环境自带的 Python 标准库 `telnetlib`。
- 放宽通过 Telnet/base64 回传配置结果时的等待时间，修复真实设备上可能出现的“回传结果超时”。
- macOS 版同步获得相同后端兼容性修复，降低系统 Python 版本变化带来的风险。

### 工程和资产整理

- 重构 App 图标资产目录，统一放到 `Assets/AppIcon/`。
- 图标源图、macOS `.icns` / iconset、Windows `.ico` 均作为设计/发布资产提交到仓库。
- 普通构建不再动态生成图标，只复制或引用已提交的图标资产。
- 删除重复的 macOS 图标资源，避免多份图标不同步。
- 新增本地 release 归档流程：本机执行 `./scripts/build_release.sh` 时，如果西数 4T NAS 已挂载，会自动复制一份 macOS 构建产物到 NAS 的项目 release 目录。
- GitHub Actions 会明确跳过本地 NAS 归档步骤，不影响云端自动构建。

### 已验证

- Windows x64：build23 经过真实设备测试通过。
- macOS arm64：build23 经过本地安装包测试通过。
- GitHub Actions 多平台构建通过。
